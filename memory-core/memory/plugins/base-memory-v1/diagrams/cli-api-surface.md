## What This Map Shows

This diagram separates the product API from the developer inspection commands. The CLI is useful while we are designing, but the system should not become CLI-shaped.

## How to read it

- The product action is `onectx.memory.hire_agent`.
- The future daemon or menu bar should call the same Python loader, account linker, memory linker, and harness code directly.
- The CLI commands are inspection doors: `show`, `map`, `harnesses`, `accounts link`, `memory`, `native-route`, `birth`, and `ledger`.
- The future harness launch step should start Codex or Claude from the runtime experience selected by the linker.

## Why it matters

This preserves the simple human action: hire this agent for this job. The command line remains a tool for us, not the product model.
