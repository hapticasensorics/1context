# Changelog

This file records every change to the wiki-engine itself (not to
content built with it). SemVer applies. Pre-1.0 = breaking changes
expected at any point.

## v0.3.0 — 2026-04-21 (current)

**Bootstrap milestone.** The engine renders its first real page —
itself. `preview/wiki-engine.html` is now generated by
`wiki-engine/src/renderer/` from `preview/public/wiki-engine.md`,
not hand-authored. The compiler that compiles the compiler.

What landed:

- Four custom directives implemented as marked extensions, one
  file each under `wiki-engine/src/renderer/directives/`:
  - `infobox.mjs` — `:::infobox … :::` → `<aside class="infobox">`
  - `main-article.mjs` — `:::main-article slug :::` → hatnote
  - `see-also.mjs` — `:::see-also … :::` → H2 + list
  - `audience.mjs` — `:::audience tier … :::` → wrapper for P5
  - shared parser at `_shared.mjs`; aggregated by `index.mjs`
- Pandoc fenced-div syntax. Closing fence matches both
  `^:::` (empty body, e.g. `:::main-article slug\n:::\n`) and
  `\n:::` (body content) — found this fix when an empty
  main-article block ate the next directive's content.
- Renderer emits `<meta name="generator" content="1Context
  wiki-engine">` so the bootstrap is provable in the HTML.
- `preview/public/wiki-engine.md` updated to use the directives:
  `:::infobox` for the fast-facts sidebar, `:::see-also` for the
  related-pages section. Hand-authored `# Wiki engine` H1 and the
  intro blockquote stripped (template emits H1 + subtitle from
  frontmatter; body now starts with the lead paragraph).

Still hand-authored: every other page in `preview/` (1context-project,
agent-ux, talk-conventions, talk-page-format). The build pipeline
that runs the renderer over every .md file is the next P1 follow-on
(see talk page topic). Until then, regenerating wiki-engine.html
is a manual `node` invocation.

## v0.2.1 — 2026-04-21

P1 renderer scaffold landed. Code under
`wiki-engine/src/renderer/`:

- `index.mjs` — `renderPage(source, { slug }) → { html, md, frontmatter }`
- `frontmatter.mjs` — gray-matter parser + 1Context-specific
  validator (required fields, allowed enums, slug format)
- `toc.mjs` — H2/H3 → themed `<nav class="opctx-toc">` block,
  preserving the existing slug convention so inbound links survive
- `template.mjs` — page-shell assembler (header, layout, link
  rel=alternate, OG meta from frontmatter)
- `directives/` — empty stubs + a README documenting the pandoc
  fenced-div pattern; first four directives (`infobox`,
  `main-article`, `see-also`, `audience`) are P1-followon TODOs

CLI smoke test: `node wiki-engine/tools/render-test.mjs <slug>`
writes to `_render-test/` (gitignored) so devs can diff against
the hand-authored version without overwriting source.

The scaffold is not yet wired into the build — every `preview/*.html`
file is still hand-authored. Verified end-to-end on every authored
page (1context-project, wiki-engine, agent-ux, talk-conventions,
talk-page-format). The renderer caught one real schema drift in
the process — `1context-project.md` had legacy `tier:`/`updated:`
shape; fixed in the same change.

New deps: `marked` ^16.x, `gray-matter` ^4.x.

## v0.2.0 — 2026-04-20

Roadmap P2 — engine code migrated into `wiki-engine/`.

- `css/` → `wiki-engine/theme/css/`
- `preview/js/` → `wiki-engine/theme/js/`
- `tools/` → `wiki-engine/tools/`
- `preview/main.js` import paths updated to reach engine assets via
  `../wiki-engine/theme/...`
- `preview/components.html`'s inline import updated similarly
- `wiki-engine/tools/*.sh` and `wiki-engine/tools/build-discovery-files.py`
  updated to compute repo root from their new location
  (`parents/../..` instead of `parents/..`)

`vite.config.mjs` and `package.json` stay at the repo root (vite
expects them there). Build verified green; discovery generator
verified green; deploy scripts updated (paths inside the scripts;
their callers should now use `./wiki-engine/tools/*.sh`).

No behavior changes. Pure code reorganization.

## v0.1.0 — 2026-04-20

Initial scope-marker release. Engine code currently lives in the
parent repo at `css/theme.css`, `preview/js/enhance.js`,
`tools/build-discovery-files.py`, `vite.config.mjs`. This folder
is established as the migration target.

What works today (in the parent dirs):
- Markdown-twin parallel files served alongside HTML
- Themed HTML rendering (hand-authored — renderer pipeline pending)
- LKML-flavored talk pages with parser + themed render
- Discovery-file generator (`llms.txt`, `llms-full.txt`, `docs-index.json`)
- Reader/Agent/Talk view toggle in header chrome
- Mobile-hardened layout (TOC drawer, scroll-direction header, copy buttons)

Known gaps:
- No MD → HTML pipeline yet (every page hand-authored). See
  ROADMAP item P1.
- No storage-adapter abstraction (everything is filesystem). See
  ROADMAP item P3.
- BookStack module artifacts (`functions.php`, `bookstack-module.json`,
  `src/Http/`, `views/`) removed in this release; the engine is no
  longer a BookStack theme.
