# Talk Conventions - Topics

Topics is the concept index. Its talk page should help the system avoid both
under-recording repeated ideas and over-promoting one-off phrases.

Use one timestamped markdown file per contribution. Each top-level file is one
thread. Replies use `parent:` frontmatter pointing to the parent filename or
stem.

## What Belongs Here

- Proposed concepts that recur across work.
- Category changes for Engineering, Infrastructure, Process, Tools, Domain,
  People, or Organizations.
- Evidence that a topic deserves a page.
- Merges when two names describe the same idea.
- Splits when one broad topic is hiding several separate ideas.

## Promotion Rule

A topic should explain future work. Prefer repeated evidence across sessions,
explicit naming by the operator, or a concept that connects several projects.

## Archive Policy

Keep active talk entries in this folder for 90 days. After that, move settled
conversation files into `archive/` while preserving filenames, frontmatter, and
parent/thread references.
