# Talk Conventions - Projects

Projects is an operational index. Its talk page should make project state
changes auditable: why something is active, paused, completed, or archived.

Use one timestamped markdown file per contribution. Each top-level file is one
thread. Replies use `parent:` frontmatter pointing to the parent filename or
stem.

## What Belongs Here

- Proposals to add or rename a project.
- Evidence that a project is active, paused, blocked, completed, or archived.
- Handoff notes that prevent future sessions from rediscovering project state.
- Cross-project patterns that may belong in Your Context or Topics.

## Promotion Rule

Do not add every repo or passing idea. A project entry should have a stable name
and enough evidence that future work benefits from a durable pointer.

## Archive Policy

Keep active talk entries in this folder for 90 days. After that, move settled
conversation files into `archive/` while preserving filenames, frontmatter, and
parent/thread references.
