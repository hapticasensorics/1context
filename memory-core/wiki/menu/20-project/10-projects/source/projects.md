# Projects

Projects is the memory system's map of active work. It should help a future
session answer: what exists, what state it is in, what changed recently, and
where the next useful action probably lives.

## Active

No active projects have been promoted yet.

For agents: add a project when it has a stable name, a working directory or
artifact trail, and enough recent evidence that future sessions would benefit
from a durable pointer.

## Paused Or Blocked

No paused or blocked projects have been promoted yet.

Record blockers when the reason matters: waiting for a person, permission,
hardware, an upstream decision, a failed build, or a design question that should
not be rediscovered.

## Recently Completed

No completed projects have been promoted yet.

Record completed work when its result is reusable: a shipped release, a fixed
integration, a migrated system, a retired experiment, or a lesson that should
shape future work.

## Archived

No archived projects have been promoted yet.

Archive projects when they are no longer active but still explain why the
current system looks the way it does.
